当前项目 (TiZi) vs Beremiz 对比分析

  当前状态

  TiZi 是一个仅支持梯形图(LD)的单文件编辑器，约890行代码，早期开发阶段。

  Beremiz 的核心界面特征

  从截图分析，Beremiz 有以下关键 UI 区域：

  ┌────────────┬─────────────────────────────────────────────────┬──────────────┐
  │    区域    │                     Beremiz                     │  TiZi 当前   │
  ├────────────┼─────────────────────────────────────────────────┼──────────────┤
  │ 左侧面板   │ 项目树（POU列表、资源、网络连接）               │ ❌ 无        │
  ├────────────┼─────────────────────────────────────────────────┼──────────────┤
  │ 右侧面板   │ 函数库浏览器 + 调试器                           │ ❌ 无        │
  ├────────────┼─────────────────────────────────────────────────┼──────────────┤
  │ 中央区域   │ MDI 多标签编辑器（LD/ST/SFC/FBD/IL）            │ 单一 LD 画布 │
  ├────────────┼─────────────────────────────────────────────────┼──────────────┤
  │ 变量声明表 │ 每个编辑器顶部有变量表（#/名称/类/类型/初始值） │ ❌ 无        │
  ├────────────┼─────────────────────────────────────────────────┼──────────────┤
  │ 底部面板   │ Search / Console / PLC Log                      │ ❌ 无        │
  ├────────────┼─────────────────────────────────────────────────┼──────────────┤
  │ 支持的语言 │ LD, ST, IL, SFC, FBD（5种）                     │ 仅 LD        │
  └────────────┴─────────────────────────────────────────────────┴──────────────┘

  需要做的改造（按优先级）

  阶段1 - 界面骨架重构（最重要）
  1. 主窗口布局改为三栏式：左侧项目树 + 中央 MDI + 右侧库面板
  2. 底部添加 Console/Log 面板（QTabWidget）
  3. 工具栏更新

  阶段2 - 项目管理
  1. 项目数据模型（POU 组织单元、资源、变量）
  2. 左侧 QTreeWidget 项目树
  3. 文件保存/加载（XML）

  阶段3 - 编辑器扩展
  1. 每个 POU 顶部的变量声明表（QTableWidget）
  2. ST 文本编辑器（最简单，用 QPlainTextEdit + 语法高亮）
  3. MDI 多标签（可同时打开多个 POU）

  阶段4 - 函数库
  1. 右侧可折叠的函数类别树
  2. 标准函数块（ADD、SEL、比较、计时器等）

  阶段5 - 高级编辑器
  1. IL 指令列表编辑器
  2. FBD 功能块图（类似 LD 的图形编辑器）
  3. SFC 顺序功能图

  ---
  你想从哪个阶段开始？建议先做阶段1的界面骨架重构，因为它能立刻让项目看起来像 Beremiz，而且不需要太多底层逻辑改动。
